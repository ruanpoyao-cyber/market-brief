# Market Brief — 互動式市場晨報

每天台北 06:30 自動產出盤後市場報告：漲幅前 30、市值增加前 30、成交金額前 30、
跌幅前 10、市值減少前 10，依族群分類做樞紐分析，標的 hover 顯示 60 日 K 線疊量，
並可依日期回看每天的報告。

> 命名刻意不綁定單一市場。**目前先實作美股**；之後要加其他市場（台股、日股…）時，
> 各市場用自己的 build_data_<market>.py 產生資料，前端再加市場切換即可，主架構不變。

## 檔案

| 檔案 | 角色 | 狀態 |
|---|---|---|
| index.html | 互動式前端成品（所有區段、hover K 線、日期切換） | 已用擬真樣本驗證 |
| make_sample.py + template.html | 產生樣本網站（前端展示用，不需連網） | 可跑 |
| build_data_us.py | 美股資料抓取（FMP），輸出 data.json | 邏輯與已驗證的抓取同型，需在有網路環境實測 |

前端已完整驗證；唯一未實測的是美股資料抓取——開發沙盒未開放 FMP/Yahoo 等網域。

## 頁面區段

1. 日期（可切換，回看每天報告）
2. 四大指數 + VIX（游標移上顯示 K 線）
3. 市場重點新聞摘要（英文來源標題一律翻為繁體中文）
4. 樞紐分析 - 漲幅前30 / 市值增加前30 / 成交金額前30（依族群）
5. 樞紐分析 - 跌幅前10 / 市值減少前10
6. 連續三天上榜標的
7. 漲幅 / 市值增加 / 成交金額表格 + 分析（含市值、市值增減、成交金額；標的可 hover K 線）
8. 跌幅 / 市值減少表格 + 分析

## 資料源（美股）

建議 FMP（Financial Modeling Prep）：一家涵蓋全市場排行、市值、類股分類、60 日 K 線、新聞。
替代：Polygon、Nasdaq 非官方端點。詳見 build_data_us.py 註解。

## 部署（建議 GitHub Pages + Actions，一天一次）

- cron "30 22 * * 1-5"（UTC = 台北 06:30）跑 build_data_us.py，commit data.json，Pages 自動更新。
- data.json 滾動保留最近 60 個報告日；要更久可改每天一個 reports/<date>.json + manifest。
- 上線時把 index.html 資料來源從內嵌樣本改為 fetch('data.json')。

## 成本

- 主機：免費。FMP：免費層或低價付費層。Gemini（新聞摘要+分析，一天一次）：每月台幣數十元等級。

## 加入新市場的方式（未來）

1. 新增 build_data_<market>.py，輸出相同 schema（沿用 build_data_us.py 為範本）。
2. 各市場各自一份 data，或合併成多市場 bundle。
3. 前端加一個市場切換（與現有日期切換同一排即可）。
